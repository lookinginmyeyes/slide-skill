# Prompt Patterns

Use these when writing actual prompts.

## Editorial Cover Pattern

```text
Purpose: opening image for a premium B2B slide deck about <topic>
Subject: <specific object/person/environment>
Composition: wide 16:9, strong negative space on <left/right>, clear focal
  object, no clutter
Style: editorial product photography, refined, not stock-like
Lighting / color: <palette>, soft contrast, controlled highlights
Text policy: no readable text, no logos, no UI labels
Output constraints: 16:9, high resolution, realistic details
Negative constraints: no generic gradient background, no floating icons, no
  fake charts, no distorted hands, no watermark
```

## Product UI Scene Pattern

```text
Purpose: visual anchor for a SaaS workflow slide
Subject: believable product interface shown on a desktop screen or floating
  panels
Composition: clean 16:9 frame, one primary panel, two secondary panels, room
  for HTML labels
Style: premium enterprise UI mockup, precise spacing, calm surfaces
Text policy: no readable body text; use abstract blocks and chart shapes only
Output constraints: UI details should be crisp but label-free
Negative constraints: no fake brand names, no lorem ipsum, no neon glow
```

## Architecture Visual Pattern

```text
Purpose: explain system flow for <system>
Subject: modular architecture with <nodes>
Composition: left-to-right flow, clear layers, few nodes, large empty margins
Style: clean technical diagram rendered as polished editorial illustration
Text policy: no readable labels; labels will be added in HTML
Output constraints: 16:9, high contrast between layers
Negative constraints: no tangled cables, no random icons, no tiny text
```
