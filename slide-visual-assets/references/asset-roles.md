# Asset Roles

## Evidence

Use when an image proves the claim. Examples:

- screenshot
- customer environment
- chart snapshot
- document excerpt
- product state

Evidence images should be captioned or annotated in HTML.

## Product Context

Use when the audience needs to understand what the product feels like. Prefer
real screenshots or faithful mockups. Avoid generic device mockups without a
specific state.

## Concept Anchor

Use when the content is abstract. One strong image can make a section memorable.
Do not overuse concept anchors on every slide.

## Process / Diagram

Use for sequences, systems, workflows, architectures, and dependencies. Labels
belong in HTML so they stay editable and readable.

## Atmosphere

Use sparingly on cover and section slides. Atmosphere should set tone, not carry
claims.
