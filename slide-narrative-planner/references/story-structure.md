# Story Structure Reference

Use this only when the deck's logic is weak or the source material is messy.

## Minimal Intake

If critical context is missing, infer safely when possible. Ask only when the
answer changes the deck substantially.

- audience
- desired decision or action
- time / slide count
- must-include points
- source reliability
- tone

## Ghost Deck Test

Write only the action titles in sequence. If they do not form a coherent
argument, revise before visual generation.

Weak:

1. Market
2. Product
3. Metrics
4. Roadmap

Better:

1. The market is shifting from manual review to assisted workflows
2. Our product wins because it shortens review time without replacing experts
3. Early usage shows the workflow is sticky in high-frequency teams
4. The next release should focus on integrations, not more dashboard features

## Evidence Ladder

Order evidence from strongest to weakest:

1. user data / measured result
2. customer quote or direct observation
3. benchmark or third-party source
4. internal expert assessment
5. hypothesis

Mark hypothesis clearly.
