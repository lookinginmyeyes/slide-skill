# Click Beat Planning

Use this for dynamic, video-like, or talk-driven HTML decks.

## Beat Rules

- One click equals one meaningful information reveal.
- A beat can reveal a phrase, metric, row, annotation, screenshot region, or
  relationship.
- Do not put all details on screen before the speaker reaches them.
- Keep prior context visible only when it helps comparison or learning.
- Do not plan exact animation too early; plan the relationship that animation
  must clarify.

## Beat Format

```text
Slide 4: The bottleneck is not model quality; it is review routing
Beat 1: show current process line
Beat 2: highlight waiting states
Beat 3: reveal 41% rework caused by routing mismatch
Beat 4: state implication: fix orchestration before adding model features
```

## When Not To Use Beats

- dense appendix slides
- print/export decks
- executive decks where audience scans without speaker
